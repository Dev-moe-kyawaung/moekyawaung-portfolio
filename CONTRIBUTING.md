# Contributing

Thanks for your interest in improving this project.

## How to contribute

1. Fork the repository.
2. Create a new branch for your change.
3. Make your updates.
4. Run the project locally and verify the build.
5. Open a pull request with a clear description.

## Development checks

Before submitting a pull request, please make sure:

- `npm install` completes successfully.
- `npm run build` succeeds.
- Any new UI matches the existing design style.
- Any new public text is clear and typo-free.
- No secrets or private values are committed.

## Commit style

Use short, descriptive commit messages. Examples:

- `feat: add new project card`
- `fix: correct contact form validation`
- `chore: update deployment config`

## Pull request guidance

A good pull request should include:

- A short summary of what changed.
- Screenshots if the UI changed.
- Notes about any build or deployment impact.
- Links to related issues if applicable.

## Code style

- Keep components small and focused.
- Prefer reusable helpers over duplicated logic.
- Follow the existing file organization.
- Keep comments minimal and useful.

## Questions

If something is unclear, open an issue first and describe the change you want to make.
